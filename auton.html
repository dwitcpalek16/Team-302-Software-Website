Autonomous Routines

  • Using text files to command the robot during autonomous period allows us to quickly and easily change what it does and fine tune the distance the robot travels

    • Changing a text file doesn't require us to recompile the robot code in order to run the robot. This means that we can change one number and completely change what the robot does instead of going into the code and changing the order of actions

  • In fact, it is quite simple to write an autonomous routine. Go to this link to learn more.

Here is an example auton file that moves the robot forward 6 feet:

  • The first number tells which subsystem to control (1 is the drive-train)

  • The second number tells which action to perform (1 is drive forward for the drive system)

  • The third number tells the drive system how far to travel (in this case it will go 6 feet forward)

  • The last number tells how fast the robot will travel (it is a max speed of up to 21)

  • Generally the format is just subsystem, inputs

  • Additionally, the drive system has a speed controller which creates the second half of a trapezoidal motion profile. This ensures that the robot stops right at its target distance.
